# Хэрэгжүүлэлтийн Төлөвлөгөө: Сүлжээний Газар (Internet Café) Удирдлагын SaaS Систем

## 1. Гүйцэтгэх хураангуй (Executive Summary)
* **Системийн зорилго:** 50-500 салбартай томоохон сүлжээ газрууд болон PC тоглоомын төвүүдийг удирдахад зориулагдсан, өргөтгөх боломжтой SaaS платформ. Энэ нь төхөөрөмжүүдийг бодит цаг хугацаанд хянах, төлбөр тооцоо бодох, санхүүгийн аудит хийх боломжийг олгоно.
* **Зорилтот хэрэглэгчид:** Тоглоомын төвийн сүлжээ операторууд, франчайз эзэмшигчид болон бие даасан газрууд.
* **Үндсэн үнэ цэнэ:** Интернет тасарсан үед ч үйл ажиллагааг тасралтгүй үргэлжлүүлэх боломж (Offline resilience), луйвраас хамгаалагдсан санхүүгийн бүртгэл, төвлөрсөн удирдлага.
* **Давуу тал:** Гибрид-клауд архитектур, үйл явдалд суурилсан (Event-sourced) тооцооллын систем, Windows-д зориулсан хамгаалагдсан агент.

## 2. Системийн хамрах хүрээ (System Scope)
* **Үндсэн функцүүд (MVP):**
    * Салбаруудыг олон түвшинд удирдах (Multi-tenant).
    * PC-г бодит цагт түгжих/нээх, сессийг хянах.
    * Урьдчилсан болон дараа төлбөрт хэтэвчний систем.
    * Тасралтгүй ажиллагааг хангах "Branch Gateway".
    * Ажилтан болон админы хянах самбар.
* **Ахисан түвшний функцүүд (2-3 дугаар үе шат):**
    * Diskless boot системүүдтэй интеграц хийх (SENET/CCBoot).
    * Төхөөрөмжийн нарийвчилсан мэдээлэл (ажиллаж буй программууд, дэлгэцийн зураг, тоглоомын статистик).
    * Захиалгын систем (Reservation/booking).
    * Хоол, ундааны худалдааны POS систем.
* **Хамрахгүй зүйлс:**
    * Өөрийн үйлдлийн систем хөгжүүлэх (Windows-ийг үндсэн систем болгон ашиглана).
    * Техник хангамжийн (Hardware) үйлдвэрлэл.

## 3. Өндөр түвшний архитектур (High-Level Architecture)
Систем нь **Hybrid Cloud + Branch Gateway + Per-PC Agent** гэсэн загварыг баримтална.
* **Frontend:** Вэбд суурилсан Админ самбар болон Windows-д суурилсан Клиент Агент.
* **Backend:** Хэрэглэгч, санхүү, тайланг хариуцсан Cloud-native микросервисүүд.
* **Өгөгдлийн сан:** PostgreSQL кластер (салбар бүрээр хуваасан - Partitioning).
* **Бодит цагийн холбоо:** WebSockets/gRPC ашиглан командыг түргэн дамжуулах.
* **Branch Gateway:** Салбар бүрт байрлах "Edge" сервер. Интернет тасарсан үед дотоод сүлжээнд сессийг удирдаж, мэдээллийг түр хадгалах үүрэгтэй.
* **Клиент Агент:** PC бүр дээр ажиллах Windows Service. PC-г түгжих, ашиглалтыг хянах үүрэгтэй.

## 4. Технологийн багц (Technology Stack)
| Давхарга | Сонгосон технологи | Сонгосон шалтгаан |
| :--- | :--- | :--- |
| **Frontend (Admin)** | Next.js (TypeScript) | Хурдан ажиллагаа, SEO, TypeScript-ийн аюулгүй байдал. |
| **Backend API** | Node.js (Fastify) | Өндөр ачаалал даах чадвар, бага хугацааны саатал. |
| **Клиент Агент** | C# / .NET | Windows API-тай шууд ажиллах боломж, Shell Launcher-ийг удирдах. |
| **Бодит цаг (Realtime)** | Socket.io (WebSockets) | Хоёр талын холболт, автоматаар дахин холбогдох механизм. |
| **Өгөгдлийн сан** | PostgreSQL | Санхүүгийн мэдээллийн найдвартай байдал (ACID), JSONB дэмжлэг. |
| **Кэш (Caching)** | Redis | Сессийн төлөв болон хурдан хандалтыг хангах. |
| **Message Queue** | RabbitMQ / NATS | Салбарын мэдээллийг клауд руу нөхөн илгээх (Backfill). |
| **Дэд бүтэц** | Kubernetes (K8s) | Өргөтгөх боломж, олон бүс нутагт байршуулах. |
| **Хяналт (Monitoring)** | Prometheus / Grafana | Хэмжигдэхүүн, алдааг хянах стандарт хэрэгсэл. |
| **Authentication** | Supabase Auth / JWT | Олон салбарт ашиглахад бэлэн, найдвартай систем. |

## 5. Үндсэн домен загварууд (Core Domain Models)
* **Branch (Салбар):** ID, эзэмшигч, нэр, цагийн бүс, gateway-ийн статус.
* **Device (Төхөөрөмж):** ID, салбарын ID, hostname, техник үзүүлэлт, статус (түгжигдсэн/ажиллаж байгаа).
* **User (Хэрэглэгч):** ID, имэйл, эрх (админ/ажилтан/хэрэглэгч), статус.
* **Wallet (Хэтэвч):** ID, хэрэглэгчийн ID, үлдэгдэл, валют.
* **Session (Сесс):** ID, хэрэглэгчийн ID, төхөөрөмжийн ID, эхэлсэн/дууссан цаг, төлөв, зардал.
* **Transaction (Гүйлгээ):** ID, хэтэвчний ID, төрөл (цэнэглэлт/зарлага), дүн, давхар гүйлгээнээс хамгаалах түлхүүр (idempotency_key).
* **PricingPlan (Үнийн тариф):** ID, салбарын ID, минут бүрийн үнэ, багц үнэ.

## 6. Хэтэвч ба Төлбөрийн логик (Wallet & Payment Logic)
* **Цэнэглэлт:** Ажилтан бэлэн мөнгөөр эсвэл хэрэглэгч онлайнаар цэнэглэнэ. Давхар гүйлгээ үүсэхээс хамгаална.
* **Төлбөр тооцоо:** 
    1. Эхлэх: PC нээх үед үлдэгдлийг шалгаж, барьцаалах эсвэл шууд хасах.
    2. Үргэлжлэх: Тодорхой хугацаанд (жишээ нь 60 секунд тутамд) салбарын сервер клауд руу мэдээлэл илгээнэ.
    3. Дуусах: Сесс дуусахад эцсийн тооцоог хийж гүйлгээг баталгаажуулна.
* **Оффлайн горим:** Интернет тасарсан үед Gateway өөр дээрээ кэшлэгдсэн үлдэгдлээр үйл ажиллагааг үргэлжлүүлнэ. Холболт сэргэхэд мэдээллийг синхрончилж, санхүүг нөхөн бодно.

## 7. API дизайн (Жишээ)
* `POST /auth/login` - Нэвтрэх.
* `GET /branches` - Салбаруудын жагсаалт.
* `POST /devices/:id/unlock` - PC-г алсаас нээх.
* `POST /wallets/topup` - Хэтэвч цэнэглэх.
* `GET /sessions/active` - Одоо тоглож буй хүмүүсийн мэдээлэл.

## 8. Клиент Агентын дизайн
* **Үүрэг:** Windows-ийн Task Manager, Ctrl+Alt+Del зэргийг хянах, PC-г бүрэн түгжих.
* **Heartbeat:** 10 секунд тутамд Gateway-д өөрийн статусыг илгээнэ.
* **Аюулгүй байдал:** Төхөөрөмжийн ID-г техник хангамжийн түвшинд (TPM/MAC) баталгаажуулж, агентыг дуурайлган үйлдэл хийхээс хамгаална.

## 9. Бодит цагийн систем
* **WebSocket:** Салбараас Клауд руу, Клаудаас Агент руу командыг 1 секундээс бага хугацаанд хүргэнэ.
* **Алдаа засалт:** Холболт тасарвал Агент автоматаар дотоод Gateway-тэй холбогдож үйл ажиллагааг үргэлжлүүлнэ.

## 10. Аюулгүй байдлын загвар
* **RBAC:** Ажилтан бүр зөвхөн өөрт зөвшөөрөгдсөн үйлдлийг хийнэ (жишээ нь: зөвхөн PC нээх, мөнгө буцааж болохгүй).
* **Audit Logging:** Санхүүгийн бүх өөрчлөлт, алсын удирдлагын команд бүрийг засварлах боломжгүйгээр бүртгэнэ.

## 11. Өргөтгөх стратеги
* **Scaling:** Клауд серверүүд ачааллаас хамааран автоматаар нэмэгдэнэ (Kubernetes HPA).
* **Multi-tenant DB:** Нэг өгөгдлийн санд олон салбарын мэдээллийг салбар бүрээр тусгаарлан (Partitioning) хадгална.

## 12. Байршуулах архитектур
* **Орчин:** Хөгжүүлэлтийн (Dev), Тестийн (Staging), Үйлдвэрлэлийн (Production) гэсэн 3 шатлалтай.
* **CI/CD:** GitHub Actions ашиглан кодыг автоматаар тестэлж, байршуулна.
* **Нөөцлөлт:** Өгөгдлийн санг цаг тутамд нөөцөлж, ямар ч үед өнгөрсөн рүү буцах (PITR) боломжтой байна.

## 13. Хяналт ба ажиглалт
* Салбар бүрийн интернет холболт, PC-үүдийн ачаалал, санхүүгийн зөрүү зэргийг хянах самбарт харуулна.
* Гэнэтийн алдаа гарвал (жишээ нь: PC түгжээгүй үед интернет тасрах) админд шууд дохио (Alert) илгээнэ.

## 14. Хөгжүүлэлтийн замын зураг (Roadmap)
* **0-р үе шат:** Суурь дэд бүтэц, өгөгдлийн сангийн схем.
* **1-р үе шат:** Санхүүгийн хэтэвч, API-ийн үндсэн функцүүд.
* **2-р үе шат:** Windows Агент болон PC түгжих/нээх механизм.
* **3-р үе шат:** Бодит цагийн удирдлага, Админ самбар.
* **4-р үе шат:** Олон салбарын удирдлага, өргөтгөл, аюулгүй байдлын хамгаалалт.

## 15. Өгөгдлийн сангийн схем
* Үндсэн хүснэгтүүд: `tenants`, `branches`, `accounts`, `wallet_events`, `sessions`, `session_events`.

## 16. Эрсдэл ба бууруулах арга хэмжээ
* **Техник эрсдэл:** Клауд сервер тасрах. *Шийдэл:* Салбар бүр дээрх Gateway ажиллагааг үргэлжлүүлнэ.
* **Аюулгүй байдлын эрсдэл:** Хэрэглэгч PC-г хакердах. *Шийдэл:* Windows Shell Launcher ашиглан программыг бүрэн тусгаарлана.

## 17. Эхний 30 хоногийн төлөвлөгөө
* **1-р долоо хоног:** Архитектур ба API контракт тодорхойлох.
* **2-р долоо хоног:** Санхүүгийн хэтэвчийн систем болон нэвтрэх хэсэг.
* **3-р долоо хоног:** PC Агентын анхны хувилбар (PC түгжих).
* **4-р долоо хоног:** Админ самбараас PC-г алсаас удирдаж, төлбөр бодож үзэх демо.
